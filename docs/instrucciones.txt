Escribe un plan detallado de implementacion para una con app mobile con las siguientes caracteristicas:

Una app en flutter enfocada en el apartado mobile (android y IOS)

debe tener las siguientes funcionalidades:
la app analizara un archivo json para extraer de el una serie de datos (ver el archivo datos.json para entender la estructura de los datos)

# Pagina Principal (home)
tendra 3 apartados Principales (que se podran dezplazar desde botones en su footer):
 - materias (izquierda)
 - disponibilidad (el home por defecto)
 - pagina profesores

 en el home se vizualizara:
 - una disponibilidad dado unos filtros (hora de inicio, hora de fin, dia de la semana, fecha (por defecto la actual), bloque, salon, incluir salones no disponibles), mostrando una tabla con los salones disponibles con esos filtros (por defecto cargar la disponibilidad con un rango de una hora, en todos los salones) -> al dar click a un salon se abrira una nueva pagina con la informacion del horario semanal completa del salon con el formato de (horario.png)

 - otra seccion donde se van estadisticas de los horarios, estadisticas de horarios por bloques (cuantos horarios con salones tiene), y estadisticas de horarios por dia (lunes, martes,...), teniendo valores como clases unicas que se dan, cuantos nrcs se dan, profesores y demas datos utiles

 # Pagina profesores:
 se puede buscar por nombre parcial un profesor, entre los profesores del filtrado el usuario le puede dar click a uno de esos y se vera su horario completo (horario semanal) con todas las materias que este da (el formato de ese horario es como se ve en la imagen (Horario.png))

 antes de hacer una busqueda cualquiera por defecto se muestra en una tabla los profesores con mas horarios unicos, nrcs, y  materias

en la pagina especifica de un profesor debe verse ademas de su horario los siguientes datos (ejemplo):
Clases 81
Materias 1
Horas/Semana 179.0
NRCs 20

 siempre que se ve un horario se puede dar clic en el para ver su informacion detallada en otra pagina (si se le da en un salon se ve el horario semanal completo de ese salon con el mismo formato de horario.png)

 la seccion final de la pagina general de profesores debe ser un select para seleccionar entre todos los profesores cual ver su informacion detallada

Pagina de Materias:
- buscar una materia como se hace con los profesores
- la pagina de una materia especifica muestra todos sus horarios semanales (con formato horario.png)
- mostrar estadisticas de las materias en general
- select para mostrar los datos de una materia especifica de entre todas ( con datos como: Materias 4
Nrcs 22
Profesores 10
Cupos 444: los cupos se calculan sumando los matriculados + los cupos)
- seccion para mostar el horario conjunto de 1 o varias materias a la vez
- enlace a una pagina para ver los horarios semanales completos de un conjunto de materias (BIO, IST, ...)
- enlace a una pagina para consultar la informacion de un nrc dado


tendremos un apartado de settings donde el usuario podra hacer:
-  cambiar entre todos los archivos json cargados seleccionando con cual quiere sacar los datos
- cargar un nuevo json (con un formato valido)

# requisitos no funcionales:
 - toda la app en modo oscuro: con una paleta de colores rojisos (tono pastel), y fondos oscuros
 - responsive para poder ver correctamente toda la UI en modo vertical o modo horizontal del dispositivo
 - debe estar todo completamente en textos en español
 - no login ni registro de usuarios
 - componetizar toda las partes de la UI para permitir reutilizacion de estilos como lo seria las cards, inputs, el componente calendario o horario


agrega en el plan lo siguiente como configuraciones del agente:
- consulta en internet dependencias de flutter que ayuden a facilitar el desarrollo (y consulta como usarlas correctamente) y acelere la velocidad de la app
- no usar emogis en ninguna parte, usar iconos cuando sea necesario y comprensible para ayuda del entendimiento de la app por parte del usuario
- implementa de manera optimizada las clases y funciones que analize los datos del json con los datos (generalmente mas de 16 mil elementos (jsons) en una lista)
- toda la informacion de horarios es referente a Uninorte
